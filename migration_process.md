# Migration process — `final.tsv` → `data/final_data/`

## Goal (current phase)
Create an updated *normalized data web* without editing `final.tsv`.

- `final.tsv` remains read-only.
- All new “final” artifacts live under `data/final_data/`.
- Denomination + sect language is replaced by a single concept: **persuasion**.

## Caveats / rules (authoritative)
- We use **persuasions** for both historic and modern affiliation labels (e.g., Montanist, Orthodox, Arian, Gnostic, Catholic).
- We collapse splinter tokens into a single persuasion where appropriate.
  - Example: all Donatist variants collapse to `donatist`.
- (Later, app migration) treat post‑Reformation affiliations as only `protestant` and `anglican` (and do not model further splinters in `final_data`).

## Stage 3 — Notes (final_data)
**Output file:** `data/final_data/notes.tsv`

- Notes have **no note slug**.
- Notes reference entities via `[[type:slug]]` in `body_md`.
- Notes are sparse (state notes only on change; evidence notes only on evidence text change).

**Status:** generated by `scripts/build_final_data_from_final.ts`.

## Stage 4 — App migration (deferred until final_data stabilizes)
- Update the app build pipeline to read from `data/final_data/*.tsv`.
- Replace any UI terminology that says “denomination/sect” with “persuasion”.
