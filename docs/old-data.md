# New Schema Layout

> Full proposed schema after applying all changes in `changes.md`.
> Sections are marked **[SOURCE]** (edit directly) or **[DERIVED]** (auto-generated by scripts, never edit).

---

## Design Principles

1. **Relations are the single source of graph truth** — `relations.tsv` is the only place cross-entity connections are stored.
2. **Every entity's geography is derivable** — person→city via relations, work→city via columns, doctrine→city via 2-hop derivation chains.
3. **Static data lives once** — city planting metadata belongs in `cities.tsv`, not repeated per decade.
4. **Derived files are always regenerated** — `places.tsv`, `entity_place_footprints.tsv`, and `note_mentions.tsv` are outputs of the derive pipeline, never edited manually.
5. **Stance is first-class** — affirms/condemns/neutral is a queryable field, not a freeform string buried in `reason`.

| `work` | `works.tsv` | `place_written_id` | `written_in` | `neutral` |

| `event` | `events.tsv` | `primary_place_id` | `held_in` | `neutral` |
| `archaeology` | `archaeology.tsv` | `city_id` → `city:{city_id}` | `located_at` | `neutral` |
| `doctrine` | relations(affirms)+works.place_written_id | chain A | `via_work_affirms` | `affirms` |
| `doctrine` | relations(condemns)+works.place_written_id | chain B | `via_work_condemns` | `condemns` |
| `doctrine` | relations(affirms)+works.place_recipient_ids | chain C | `via_work_sent_affirms` | `affirms` |
| `doctrine` | relations(condemns)+works.place_recipient_ids | chain D | `via_work_sent_condemns` | `condemns` |
| `doctrine` | relations(affirms, person)+person→city | chain E | `via_person_affirms` | `affirms` |
| `doctrine` | relations(condemns, person)+person→city | chain F | `via_person_condemns` | `condemns` |

## How the Doctrine City Map Feature Works

With this schema:

1. User selects doctrine D (e.g., "infant-baptism").
2. Query `entity_place_footprints` WHERE `entity_type='doctrine'` AND `entity_id='infant-baptism'`.
3. Group results by `place_id`, collecting `stance` values per city.
4. Color cities on the map:
   - **Green** — at least one `affirms` footprint, no `condemns`
   - **Red** — at least one `condemns` footprint, no `affirms`
   - **Yellow/striped** — both `affirms` and `condemns` footprints (contested)
   - **Grey** — no footprint (unknown / assumed)
5. Tooltip shows: person or work name (from `reason`), year range, and certainty (from the source relation).

**Zero extra manual data entry required.** Once you add a `relations.tsv` row `person:tertullian → condemns → doctrine:infant-baptism`, the derive script automatically creates footprints for every city Tertullian was `bishop_of` or `active_in`, with `stance=condemns`.

## Updated Workflow

### Derive commands
```
npm run data:derive-places      # cities + archaeology → places.tsv (existing)
npm run data:derive-footprints  # all sources → entity_place_footprints.tsv (NEW)
npm run data:derive-mentions    # notes body_md → note_mentions.tsv (NEW)
npm run data:validate           # runs all three derives, then validates FKs/headers
```

### Data-entry mental model
> **"If it's a connection between entities, it goes in `relations.tsv`. Everything else is automatic."**

| I want to... | I edit... | What auto-derives... |
|---|---|---|
| Record Ignatius was bishop of Antioch | `relations.tsv`: `person:ignatius → bishop_of → city:antioch-antakya` | footprint: person `ignatius` → `city:antioch-antakya` |
| Record that Ignatius's letters affirm monarchical episcopacy | `relations.tsv`: `work:ignatius-epistles → affirms → doctrine:monarchical-episcopacy` | footprint: doctrine `monarchical-episcopacy` → `city:antioch-antakya`, stance=affirms |
| Record Origen condemned adoptionism | `relations.tsv`: `person:origen → condemns → doctrine:adoptionist-monarchianism` | footprint: doctrine → every city Origen was active_in (Caesarea, Alexandria, Tyre...), stance=condemns |
| Add a note mentioning Paul | `notes.tsv`: body contains `[[person:paul]]` | `note_mentions.tsv`: note → person:paul |
